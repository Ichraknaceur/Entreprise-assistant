# Change Management Policy

## Purpose
This policy ensures changes to systems, applications, and infrastructure are assessed, approved, tested, and implemented in a controlled manner.

## Change Types
Changes are categorized as:
- Standard: low-risk, pre-approved
- Normal: planned and reviewed
- Emergency: urgent changes for business continuity

## Required Information
Each change request must include scope, risk, rollback plan, testing evidence, implementation schedule, and business impact.

## Approval
Normal and Emergency changes require formal approval by authorized stakeholders. Standard changes follow predefined procedures.

## Deployment Windows
Changes to critical systems must be scheduled during approved maintenance windows whenever possible.

## Post-Implementation Review
Significant changes must be reviewed after implementation to confirm expected outcomes and identify issues.
